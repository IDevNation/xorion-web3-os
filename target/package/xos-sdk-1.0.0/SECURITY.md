# Security Policy

## Supported Versions

We release patches for security vulnerabilities regularly. The following versions are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Xorion seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email at **security@x-os.network** with the following information:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (if applicable)
- Your contact information for follow-up questions

You should receive a response within **48 hours** confirming receipt of your report.

## Disclosure Policy

Once we have received your report, we will:

1. Acknowledge receipt within 48 hours
2. Investigate and validate the reported issue
3. Develop a fix and test it thoroughly
4. Release a patched version
5. Publicly disclose the vulnerability after users have had time to update

We request that you keep the vulnerability confidential until we have released a patch and publicly disclosed the issue.

## Response Timeline

| Stage | Timeline |
|-------|----------|
| Initial Response | Within 48 hours |
| Validation | 1-7 days |
| Fix Development | 1-14 days |
| Patch Release | As soon as testing is complete |
| Public Disclosure | After patch release |

## Security Best Practices

When using Xorion, please follow these security best practices:

- Keep your private keys secure and never share them
- Use hardware wallets for large amounts
- Regularly update to the latest version
- Enable all available security features
- Review transaction details before signing

## Recognition

We appreciate responsible disclosure and will acknowledge researchers who report valid security issues (unless they prefer to remain anonymous).

Thank you for helping keep Xorion and our users safe!
